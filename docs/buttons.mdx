# buttons

## primary

Primary buttons are medium sized and have a colored background. They aren't always blue, though, so there isn't a default "primary" look or CSS class - primary here is more of a concept.

Please note that the color and size are built through CSS classes applied to the `<button>` element. You'll always need to specify color and size to get a complete button.

Use these for the main action in a section or screen. You should display only one primary button at a time.

<Playground>
  <button className="btn btn--uranus btn--medium">
    primary
  </button>
  
  <button className="btn btn--uranus btn--medium" disabled>
    primary disabled
  </button>
</Playground>

## colors

Buttons can have different colors simply by changing their CSS color class.

<Playground>
  <button className="btn btn--uranus btn--medium">
    uranus
  </button>
 
  <button className="btn btn--venus btn--medium">
    venus
  </button>

  <button className="btn btn--earth btn--medium">
    earth
  </button>
</Playground>

## sizes

Buttons can have different sizes, also by changing their CSS size class.

<Playground>
  <button className="btn btn--uranus btn--small">
    small
  </button>
 
  <button className="btn btn--uranus btn--medium">
    medium
  </button>

  <button className="btn btn--uranus btn--large">
    large
  </button>
</Playground>

## outline

Outline buttons can be used as secondary roles. Secondary buttons present secondary actions when there's already a primary action in the same section or screen.

Please note that there are color variations to the outline CSS classes. You also need to specify their size with one of the size CSS classes.

<Playground>
  <button className="btn btn--outline-uranus btn--small">
    outline uranus
  </button>
  <button className="btn btn--outline-uranus btn--medium">
    outline uranus
  </button>
  <button className="btn btn--outline-uranus btn--large">
    outline uranus
  </button>
</Playground>

## `<a>` elements

Although the button classes are designed for `<button>` elements, you may also use them with `<a>` elements when it's semantically recomended. Note that some browsers may render them slightly differently. 

`<a>` elements don't have a `:disabled` attribute/pseudo selector, so if you need that state, you need to add a `disabled` class like the example below.

If your `<a>` button triggers in-page functionalities instead of linking (to other pages or within the current page), make them semantically accessible by adding a `role="button"`.

<Playground>
  <a className="btn btn--uranus btn--medium">
    link
  </a>
  <a className="btn btn--uranus btn--medium disabled">
    link disabled
  </a>
</Playground>
