# inputs

#### Inputs appear when the user needs to input data. They are mostly used in forms and in contexts that require data such as money, text and numbers.

## standard text inputs

The regular input. Labels appear on the default state and slightly move upwards when the user starts interacting and typing on the component.

Please follow the markup structure below in the same order to get all correct CSS styles.

The floating label will need JS to work on your project. You may find the code for this working example in our GitHub repository.

To create the disabled state, add the `a-input--disabled` modifier class on element root and `disabled` attribute on the `<input>` tag.

## masked inputs

Inputs that have a specific value structure for certain kinds of data.

Masked inputs don't need specific CSS classes other than the main one shown below. However, they do need a [mask plugin](https://github.com/text-mask/text-mask) and custom JS to work on your project. You may find the code for this working example in our GitHub repository. If you decide to use Text Mask like we did, please follow the example to get the same results.

<Playground className='md:two-columns'>
  <div className='a-input'>
    <input
      id='inputdate'
      type='text'
      placeholder='DD/MM/YYYY'
      aria-labelledby='labeldate'
    />
    <label id='labeldate' htmlFor='inputdate'>
      Date
    </label>
  </div>
  <div className='a-input'>
    <input
      id='inputmoney'

      aria-labelledby='labelcpf'
    />
    <label id='labelcpf' htmlFor='inputcpf'>
      CPF

## ghost inputs

Ghost inputs are used in the foreground of colorful backgrounds or images, always displayed in white color in order to preserve good usability and legibility. Depending on the context, when the user starts typing, a white icon might appear on the right side of the component, fading in.

The floating label will need JS to work on your project. You may find the code for this working example in our GitHub repository.

This input style does not have a disabled state.

## control inputs

These inputs are commonly used for money input data, in contexts that require increasing and decreasing the value in large amounts while still enabling users to type in numbers manually.

They carry outline [icon buttons](https://astro.magnetis.com.br/buttons#icon-buttons) and these will always add or subtract R$ 100 or R$ 1.000.

The buttons will need JS to work on your project. You may find the code for this working example in our GitHub repository. We also used a plugin for the currency mask, as explained in the Masked Inputs section above.

To create the disabled state, add the `a-input--disabled` modifier class on element root and `disabled` attribute on the `<input>` tag.

      Control input (R$)

    </button>
    <button className='a-input__btn-control'>
      <i className='a-icon a-icon--input-plus a-icon--size-small' />
    </button>

## messaging inputs

Inputs used for messaging and similar contexts. They carry embedded [icon buttons](https://astro.magnetis.com.br/buttons#icon-buttons) that send the data,
and their color can be changed as explained in the [Buttons](https://astro.magnetis.com.br/buttons) page.

Click inside to view `:focus` state.

The button color/state logic will need JS to work on your project. You may find the code for this working example in our GitHub repository.

To create the disabled state, add the `a-input--disabled` modifier class on element root and`disabled` attribute on the `<input>` and `<button>` tags.

<Playground className='md:two-columns'>
  <div className='a-input a-input--messaging'>

    <button className='a-btn a-btn--uranus a-btn--medium a-btn--icon' disabled>
      <i className='a-icon a-icon--arrow-right a-icon--size-medium' />

      <i className='a-icon a-icon--arrow-right a-icon--size-medium' />

</Playground>
